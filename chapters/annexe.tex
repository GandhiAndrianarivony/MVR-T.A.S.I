\annexechapter{Librairies}
Cette annexe présente l'ensemble des bibliothèques utilisées tout au long de ce travail. L'environnement de développement repose sur des librairies scientifiques fondamentales telles que \texttt{NumPy}, \texttt{SciPy} et \texttt{Pandas} pour le calcul numérique et la manipulation de données. Pour les opérations d'imagerie, nous utilisons \texttt{scikit-image} pour les transformations géométriques et les métriques d'évaluation, \texttt{PyWavelets} et \texttt{ptwt} (PyTorch Wavelets) pour les décompositions en ondelettes sur CPU et GPU respectivement. La gestion des images est assurée par \texttt{PIL}, tandis que \texttt{tqdm} fournit les barres de progression pour le suivi des calculs longs.

L'implémentation des algorithmes de reconstruction exploite plusieurs bibliothèques spécialisées. \texttt{PyLops} est utilisé pour la définition d'opérateurs linéaires et l'implémentation des algorithmes ISTA et FISTA. La bibliothèque \texttt{QBI\_radon} offre une alternative pour la transformation de Radon. Pour l'accélération matérielle, \texttt{CuPy} permet d'exécuter les calculs sur GPU en reproduisant l'interface de NumPy, tandis que \texttt{cupyx.scipy.sparse.linalg} fournit les solveurs itératifs optimisés pour les architectures parallèles.

L'environnement est configuré pour une exécution hybride CPU/GPU, avec une détection automatique de la disponibilité CUDA. La reproductibilité des résultats est assurée par la fixation de la graine aléatoire de NumPy. Cette architecture logicielle permet ainsi de comparer équitablement les performances des différentes méthodes de reconstruction sur les deux types de processeurs.

\annexechapter{Génération de sinogrammes}
Ce module implémente les fonctions de simulation des acquisitions tomographiques. La fonction \texttt{generate\_sinogram} réalise la transformation de Radon d'une image d'entrée selon un nombre spécifié d'angles équidistants sur 180 degrés. Elle offre également la possibilité d'ajouter un bruit gaussien paramétrable, permettant ainsi de simuler différentes conditions d'acquisition allant du cas idéal aux situations cliniques réalistes où le bruit de mesure est présent.

La fonction \texttt{generate\_undersampled\_sinogram} constitue un élément central pour l'évaluation des méthodes de reconstruction parcimonieuse. Elle simule une acquisition à faible dose en sélectionnant aléatoirement un sous-ensemble d'angles de projection, contrôlé par le paramètre \texttt{undersampling\_factor}. Cette approche reproduit fidèlement les protocoles cliniques visant à réduire l'exposition du patient en limitant le nombre de vues acquises.

Ces deux fonctions retournent non seulement le sinogramme résultant mais également les angles de projection utilisés, essentiels pour les étapes de reconstruction ultérieures. La sélection aléatoire des angles, combinée à la fixation de la graine aléatoire, garantit la reproductibilité des expériences tout en préservant la variabilité nécessaire à une validation robuste.

\begin{minted}{python}
    def generate_sinogram(
        image: np.ndarray,
        add_noise: bool =False,
        noise_level: float =0.05,
        num_angles: int= 360,
    ):
        """
        Generate sinogram from image using Radon transform
        """
        # Create equally spaced angles
        theta = np.linspace(0., 180., num_angles, endpoint=False)

        # Compute Radon transform
        sinogram = radon(image, theta=theta, circle=True)

        # Add noise if required
        if add_noise:
            noise = np.random.normal(0, noise_level * np.max(sinogram), sinogram.shape)
            sinogram = sinogram + noise

        return sinogram, theta


    def generate_undersampled_sinogram(image: np.ndarray, undersampling_factor: float=0.5):
        """
        Generate undersampled sinogram for CS reconstruction
        This function simulates a low-dose / sparse-view CT acquisition
        """
        # Full set of angles
        full_angles = np.linspace(0., 180., 360, endpoint=False)

        # Select subset of angles
        num_angles = int(len(full_angles) * undersampling_factor)
        selected_indices = np.sort(np.random.choice(len(full_angles), num_angles, replace=False))
        theta_undersampled = full_angles[selected_indices]

        # Generate sinogram only at selected angles
        sinogram = radon(image, theta=theta_undersampled, circle=True)

        return sinogram, theta_undersampled, selected_indices
\end{minted}

\annexechapter{Configuration}

Ce fichier centralise l'ensemble des paramètres de configuration de l'étude, garantissant la cohérence et la reproductibilité des expériences. La constante \texttt{DEVICE} détecte automatiquement la présence d'un GPU compatible CUDA pour basculer entre les implémentations CPU et GPU. Les dimensions des images sont fixées à $256 \times 256$ pixels, et le nombre d'itérations maximum pour les algorithmes itératifs est défini à $100$.

Les classes de configuration sous forme de \texttt{dataclasses} encapsulent les hyperparamètres spécifiques à chaque algorithme. \texttt{ADMMConfig} regroupe le coefficient de régularisation $L_1$ (\texttt{lambda\_l1}), le paramètre de pénalité (\texttt{rho}) et le type d'ondelette. \texttt{ISTAConfig} et \texttt{FISTAConfig} partagent une structure similaire avec l'ajout du paramètre de pas de descente. Enfin, \texttt{FBPConfig} définit le type de filtre pour la rétroprojection filtrée parmi les options classiques.

La classe abstraite \texttt{CTAlgorithm} établit une interface commune pour tous les algorithmes de reconstruction, imposant l'implémentation d'une méthode \texttt{process\_image}. Cette architecture orientée objet facilite l'extension du framework à de nouvelles méthodes et garantit une intégration homogène dans la chaîne d'évaluation.

\annexechapter{Opérateur de Transformée en Ondelettes}

Cette classe \texttt{WaveletOperatorGPU} implémente un opérateur linéaire pour la transformation en ondelettes 2D sur GPU, héritant de \texttt{cupyx.scipy.sparse.linalg.LinearOperator}. Elle permet de représenter la transformation comme une matrice, facilitant son utilisation dans les algorithmes d'optimisation itératifs. Le constructeur initialise la structure de décomposition en ondelettes biorthogonales Bior4.4 sur quatre niveaux et calcule la dimension de l'espace des coefficients via la méthode interne \texttt{\_count\_coeffs}.

L'opérateur définit les méthodes \texttt{\_matvec} et \texttt{\_rmatvec} qui implémentent respectivement la transformation directe (image $\rightarrow$ coefficients) et adjointe (coefficients $\rightarrow$ image). La transformation directe utilise \texttt{ptwt.wavedec2} pour décomposer l'image en coefficients d'approximation et de détails, puis aplatit le résultat avec \texttt{\_flatten}. La transformation adjointe reconstruit l'image à partir des coefficients via \texttt{ptwt.waverec2}.

La structure de l'opérateur est construite dynamiquement à partir d'une image factice, permettant de déterminer précisément la forme de chaque sous-bande de coefficients. Les méthodes \texttt{\_flatten} et \texttt{\_unflatten} assurent la conversion bidirectionnelle entre la structure hiérarchique des coefficients et un vecteur plat.

\begin{minted}{python}
    #@title WOP-GPU
    class WaveletOperatorGPU(cupyLinearOperator):
        def __init__(
            self,
            image_shape: tuple[int, int],
            wavelet: str = "bior4.4",
            level: int = 4,
            device: str = "cuda",
        ):
            self.image_shape = tuple(image_shape)
            self.wavelet = wavelet
            self.level = level
            self.device = device

            self.height, self.width = self.image_shape

            self.structure = self._build_structure()

            super().__init__(
                dtype='float32',
                shape=(self._count_coeffs(), np.prod(self.image_shape))
            )

        # ---------------------------
        # Internal utilities
        # ---------------------------
        def _build_structure(self) -> list:
            dummy = torch.zeros(self.image_shape, device=self.device)
            coeffs = ptwt.wavedec2(dummy, self.wavelet, level=self.level)

            structure = []

            structure.append(coeffs[0].shape)

            for (cH, cV, cD) in coeffs[1:]:
                structure.append((cH.shape, cV.shape, cD.shape))

            return structure

        def _count_coeffs(self) -> int:
            total = torch.prod(torch.tensor(self.structure[0])).item()
            for shapes in self.structure[1:]:
                for s in shapes:
                    total += torch.prod(torch.tensor(s)).item()
            return int(total)

        def _flatten(self, coeffs: list) -> torch.Tensor:
            flat = []

            flat.append(coeffs[0].reshape(-1))

            for (cH, cV, cD) in coeffs[1:]:
                flat.extend([
                    cH.reshape(-1),
                    cV.reshape(-1),
                    cD.reshape(-1)
                ])

            return torch.cat(flat)

        def _unflatten(self, flat: torch.Tensor) -> list:
            coeffs = []
            idx = 0

            cA_shape = self.structure[0]
            size = torch.prod(torch.tensor(cA_shape)).item()
            cA = flat[idx:idx+size].reshape(cA_shape)
            idx += size
            coeffs.append(cA)

            for shapes in self.structure[1:]:
                level_coeffs = []
                for shape in shapes:
                    size = torch.prod(torch.tensor(shape)).item()
                    c = flat[idx:idx+size].reshape(shape)
                    idx += size
                    level_coeffs.append(c)
                coeffs.append(tuple(level_coeffs))

            return coeffs

        # ---------------------------
        # Linear operator interface
        # ---------------------------
        def _matvec(
            self,
            image: Annotated[cp.ndarray, "Expected shape [N,1] ou [N,]"]
        ) -> cp.ndarray:

            img = image.reshape(self.image_shape)
            img_tensor = torch.as_tensor(img, device=self.device).float()

            coeffs = ptwt.wavedec2(img_tensor, self.wavelet, level=self.level)
            flat = self._flatten(coeffs)

            return cp.asarray(flat)

        def _rmatvec(
            self,
            w: Annotated[cp.ndarray, "Expected shape [M,1] ou [M,]"]
        ) -> cp.ndarray:

            w_tensor = torch.as_tensor(w, device=self.device).float()
            coeffs = self._unflatten(w_tensor)
            img = ptwt.waverec2(coeffs, self.wavelet)

            img = img[:self.height, :self.width]

            return cp.asarray(img.reshape(-1))

\annexechapter{Opérateur de Transformée de Radon}

Deux versions de l'opérateur de Radon sont proposées pour permettre une évaluation comparative sur CPU et GPU. \texttt{RadonOperatorCPU} hérite de \texttt{pylops.LinearOperator} et utilise les implémentations CPU de \texttt{scikit-image} pour la transformation directe (\texttt{radon}) et adjointe (\texttt{iradon} sans filtrage).

\texttt{RadonOperatorGPU} étend cette approche en exploitant \texttt{CuPy} pour les calculs sur GPU. Cette architecture hybride permet de bénéficier de l'accélération GPU tout en conservant la robustesse des implémentations CPU.

Les deux classes partagent une interface commune, ce qui les rend interchangeables dans les algorithmes de reconstruction. Leur constructeur assure la cohérence dimensionnelle avec le problème inverse formulé comme :
\[
g = Hf
\]
\begin{minted}{python}
    #@title ROP-CPU
    class RadonOperatorCPU(pylops.LinearOperator):
        def __init__(self, image_shape, theta):
            self.image_shape = image_shape
            self.theta = theta

            self.height, self.width = image_shape
            self.image_shape = (self.height, self.width)

            # forward size
            test = np.zeros(self.image_shape)
            sino = radon(test, theta=self.theta, circle=True)

            super().__init__(
                dtype='float32',
                shape=(sino.size, self.height * self.width)
            )

        def _matvec(self, x):
            x_img = x.reshape(self.image_shape)
            sino = radon(x_img, theta=self.theta, circle=True)
            return sino.ravel()

        def _rmatvec(self, y):
            sino = y.reshape(-1, len(self.theta))
            # unfiltered backprojection (adjoint)
            backproj = iradon(
                sino,
                theta=self.theta,
                filter_name=None,
                circle=True
            )
            return backproj.ravel()

    #@title ROP-GPU
    class RadonOperatorGPU(pylops.LinearOperator):
        def __init__(self, image_shape, theta):
            self.image_shape = image_shape
            self.theta = theta

            self.height, self.width = image_shape
            self.image_shape = (self.height, self.width)

            # forward size
            test = np.zeros(self.image_shape)
            sino = radon(test, theta=self.theta, circle=True)

            super().__init__(
                dtype='float32',
                shape=(sino.size, self.height * self.width)
            )

        def _matvec(self, x: Annotated[np.ndarray, "Flatten Vector"])-> cp.ndarray:
            x_img = x.reshape(self.image_shape)
            sino = radon(x_img, theta=self.theta, circle=True)
            res = cp.asarray(sino.ravel())
            return res

        def _rmatvec(self, y: Annotated[cp.ndarray, "Flatten Vector"])-> cp.ndarray:
            sino = y.reshape(-1, len(self.theta))
            sino = cp.asnumpy(sino)
            # unfiltered backprojection (adjoint)
            backproj = iradon(
                sino,
                theta=self.theta,
                filter_name=None,
                circle=True
            )
            return cp.asarray(backproj.ravel())

\annexechapter{Algorithmes}

Ce module contient les implémentations des principaux algorithmes de reconstruction. La classe \texttt{ADMMGPU} implémente l'algorithme \emph{Alternating Direction Method of Multipliers} avec accélération GPU. Elle repose sur trois étapes itératives : mise à jour de l'image via un gradient conjugué, mise à jour des coefficients par seuillage doux, et mise à jour du multiplicateur de Lagrange.

Les classes \texttt{ADMMCPU}, \texttt{ISTA}, \texttt{FISTA} et \texttt{FBP} fournissent des implémentations de référence. \texttt{ISTA} et \texttt{FISTA} s'appuient sur \texttt{PyLops}, tandis que \texttt{FBP} utilise la rétroprojection filtrée de \texttt{scikit-image}.

Chaque algorithme expose une interface unifiée via la méthode \texttt{\_\_call\_\_}, facilitant leur comparaison.

\begin{minted}{python}
    #@title ADMMGPU
    class ADMMGPU:
        def __init__(
            self,
            radon_op: RadonOperatorGPU,
            wavelet_op: WaveletOperatorGPU,
            config: ADMMConfig = ADMMConfig(),
            *args, **kwargs
        ):
            self.config = config
            self.wavelet_op = wavelet_op
            self.radon_op = radon_op

        def _lhs(self, v: Annotated[cp.ndarray, "Expected shape [N,1] ou [N,]"]) -> cp.ndarray:
            v_np: np.ndarray = cp.asnumpy(v)

            AtA: cp.ndarray = self.radon_op.rmatvec(self.radon_op.matvec(v_np))
            WtW: cp.ndarray = self.wavelet_op.rmatvec(self.wavelet_op.matvec(v))
            return AtA + self.config.rho * WtW

        def _x_update(self, x, rhs, maxiter: int = 20)-> tuple[cp.ndarray, Any]:
            """Solve linear system by using conjugate gradient"""
            linop = cupyLinearOperator((N_PIXELS, N_PIXELS), matvec=self._lhs)
            x, info = cupy_cg(
                linop,
                rhs,
                x0=x,
                maxiter=maxiter
            )
            return x, info

        def _z_update(self,x, u, z)-> cp.ndarray:
            # ---- z-update (soft threshold) ---- #
            Wx: cp.ndarray = self.wavelet_op.matvec(x)
            v = Wx + u
            thresh = self.config.lambda_l1 / self.config.rho
            z = cp.sign(v) * cp.maximum(cp.abs(v) - thresh, 0)
            return z

        def _u_update(self, x: cp.ndarray, u: cp.ndarray, z: cp.ndarray) -> cp.ndarray:
            Wx = self.wavelet_op.matvec(x)
            return u + Wx - z

        def __call__(self, sinogram: Annotated[np.ndarray, "H x W"]):
            sinogram = cp.asarray(sinogram.ravel())
            # ---- Initialize (GPU) ---- #
            x = cp.zeros(N_PIXELS, dtype=np.float32)
            z = cp.zeros(self.wavelet_op.shape[0], dtype=cp.float32)
            u = cp.zeros_like(z)

            # sparse_images = []
            primal_residuals = []
            dual_residuals = []

            for k in range(MAX_ITER):
                rhs: cp.ndarray = self.radon_op.rmatvec(sinogram) + self.config.rho * self.wavelet_op.rmatvec(z - u)

                # ---- x-update ---- #
                x, info = self._x_update(x, rhs)

                # ---- z-update (soft threshold) ---- #
                z_old = z.copy()
                z = self._z_update(x, u, z)

                # ---- u-update ---- #
                u = self._u_update(x, u, z)

                # ---- convergence ---- #
                Wx = self.wavelet_op.matvec(x)
                r_norm = cp.linalg.norm(Wx - z).item()
                s_norm = (self.config.rho * cp.linalg.norm(z - z_old)).item()

                primal_residuals.append(r_norm)
                dual_residuals.append(s_norm)

                sparsity = cp.mean(cp.abs(z) < 1e-6).item()

                if r_norm < 1e-4 and s_norm < 1e-4:
                    print(f"ADMM converged at iteration {k}")
                    break

            return (
                cp.asnumpy(x).reshape(IMAGE_SHAPE),
                # sparse_images,
                primal_residuals,
                dual_residuals,
                cp.asnumpy(z)
            )
    #@title ADMMCPU


    class ADMMCPU:
        def __init__(
            self,
            radon_op: RadonOperatorCPU,
            wavelet_op,
            config: ADMMConfig = ADMMConfig(),
        ):
            self.config = config
            self.wavelet_op = wavelet_op
            self.radon_op = radon_op

        def _lhs(self, v):
            AtA = self.radon_op.rmatvec(self.radon_op.matvec(v))
            WtW = self.wavelet_op.rmatvec(self.wavelet_op.matvec(v))
            return AtA + self.config.rho * WtW

        def _rhs(self, sinogram, u, z):
            # ---- x-update ----
            At = self.radon_op.rmatvec(sinogram)
            Wt = self.wavelet_op.rmatvec(z - u)
            return At + self.config.rho * Wt


        def _x_update(self, x, u, z, rhs, lhs, maxiter: int=50)-> tuple[np.ndarray, Any]:
            linop = LinearOperator((N_PIXELS, N_PIXELS), matvec=lhs)
            return cg(
                linop,
                rhs, x0=x, maxiter=50
            )

        def _z_update(self, x, u) -> np.ndarray:
            Wx = self.wavelet_op.matvec(x)
            v = Wx + u
            return np.sign(v) * np.maximum(np.abs(v) - self.config.lambda_l1 / self.config.rho, 0)

        def _u_update(self, x, u, z) -> np.ndarray:
            Wx = self.wavelet_op.matvec(x)
            return u + Wx - z

        def __call__(self, sinogram: Annotated[np.ndarray, "flatten vector"])-> tuple[np.ndarray, list[np.ndarray], list, np.ndarray]:
            # Initialize
            x = np.zeros(N_PIXELS)
            z = np.zeros(self.wavelet_op.shape[0])
            u = np.zeros_like(z)

            sparse_images = []
            primal_residuals = []
            dual_residuals = []

            pbar = tqdm(range(MAX_ITER), desc="ADMM", leave=True)

            for k in pbar:
                # ---- x-update ----
                x, info = self._x_update(
                    x, u, z,
                    rhs=self._rhs(sinogram, u, z),
                    lhs=lambda v: self._lhs(v),
                    maxiter=50
                )

                sparse_images.append(
                    x.reshape(IMAGE_SHAPE).astype(np.float32).copy()
                )

                # ---- z-update (soft thresholding) ----
                z_old = z.copy()
                z = self._z_update(x, u)

                # ---- u-update ----
                u = self._u_update(x, u, z)

                # ---- convergence check ----
                Wx = self.wavelet_op.matvec(x)
                r_norm = np.linalg.norm(Wx - z)
                s_norm = self.config.rho * np.linalg.norm(z - z_old)

                primal_residuals.append(r_norm)
                dual_residuals.append(s_norm)

                # ---- sparsity metric ----
                sparsity = np.mean(np.abs(z) < 1e-6)


                # ---- update tqdm ----
                pbar.set_postfix({
                    "r": f"{r_norm:.2e}",
                    "s": f"{s_norm:.2e}",
                    "ρ": self.config.rho,
                    "sp": f"{sparsity:.2%}",
                    "cg": info  # 0 = success
                })

                if r_norm < 1e-4 and s_norm < 1e-4:
                    print(f"ADMM converged at iteration {k}")
                    break

            return x.reshape(IMAGE_SHAPE), sparse_images, primal_residuals, dual_residuals, z

\annexechapter{Reconstructions}

Ce script réalise l'évaluation systématique des algorithmes. La fonction \texttt{evaluate} calcule les métriques PSNR (\emph{Peak Signal-to-Noise Ratio}) et SSIM (\emph{Structural Similarity Index Measure}) entre l'image originale et l'image reconstruite.

La boucle principale itère sur chaque algorithme puis sur chaque image. Pour chaque cas, un sinogramme sous-échantillonné est généré avec un facteur de $40\%$. Les opérateurs sont instanciés et la reconstruction est effectuée.

Les résultats sont collectés dans une structure \texttt{pandas DataFrame} pour analyse. Cette méthodologie garantit la reproductibilité et la comparabilité des performances.
